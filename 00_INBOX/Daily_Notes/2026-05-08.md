# Daily Note — 2026-05-08

### Software Engineering Doc Types — BRD, PRD, SRS, API Ref, Architecture, Deployment Guide
- Mapping document types → audience, kapan buat, tujuan
- Tabel ringkas: BRD (stakeholder, sebelum kickoff), PRD (PM+Eng, sebelum sprint), SRS (Eng, sebelum+sesudah code), API Ref (sesudah), Architecture (sebelum design+sesudah), Deployment (sesudah prod ready)
- Rekomendasi Brajakara workflow:
  - Kickoff: Client brief → BRD ✅
  - Sprint: PRD outline + SRS sketch
  - Coding: inline comment + commit message
  - Review: SRS final + Architecture doc
  - Release: API Reference + Deployment guide
  - Prod: User Manual
- Paling sering lo bikin: **SRS** (spec teknis) + **API Reference** (endpoint list)
