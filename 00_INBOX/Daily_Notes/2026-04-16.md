### Insiden: VM Tidak Bisa Konek ke Network
- **Root cause:** Nama interface salah — diasumsikan `enp0s16`, padahal aslinya `enp6s18`
- **Deteksi:** `ip link` output tunjukkan baris `2: enp6s18: <BROADCAST,MULTICAST,UP,LOWER_UP>`
- **Fix sementara (runtime):**
  ```bash
  ip addr add 192.168.1.100/24 dev enp6s18
  ip route add default via 192.168.1.1
  ```
- **Verifikasi:** `ip addr show enp6s18` → `inet 192.168.1.100/24 scope global enp6s18` ✓
- **Ping test:** `ping 8.8.8.8` berhasil (17.6ms), 75% packet loss awal tapi konek
- **Fix permanent via netplan** (`/etc/netplan/01-netcfg.yaml`):
  ```yaml
  network:
    version: 2
    ethernets:
      enp6s18:
        dhcp4: true
  ```
  lalu `netplan apply`
