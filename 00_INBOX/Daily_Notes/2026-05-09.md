# Daily Note — 2026-05-09

### [Authentik Outline Integration — Direct Google OAuth Redirect Challenge]

**Goal:** User login dari Outline → redirect langsung ke Google OAuth, bypass Authentik login page. User account tetap dimanage di Authentik.

**Progress:**
- SSH ke tower (10.20.0.11) — Authentik containers all healthy ✓
- Cek Authentik setup: v2024.10.1, docker-compose dengan PostgreSQL + Redis ✓
- Review default-authentication-flow: 4 stages (Identification → Password → MFA → UserLogin)
- Explored solutions:
  1. **Source Stage** — requires Enterprise license (not active)
  2. **Custom Template Redirect** — requires deep template override (risky)
  3. **Launch URL** — not applicable (launches from Authentik dashboard, not from external app)

**Current Blocker:**
- Authentik free version ga punya built-in "skip identification stage, direct to single social source" feature
- Feature ini ada di Enterprise (Source Stage) tapi license ga aktif

**Pain point discovered:**
- Authentik seems over-engineered untuk use-case simple (just OAuth connector + user management)
- Compared to Dex: Dex pure connector, ga ada UI bloat, but no built-in user management dashboard
- Solution: consider alternative SSO tools yang lebih lightweight + free

**Alternatives evaluated:**
1. **Keycloak** — mature, tapi berat (Java)
2. **Zitadel** — modern, Go-based, lightweight, user mgmt UI — but licensing status for self-hosted belum 100% confirmed (need to verify)
3. **Casdoor** — Go-based, open source, user management + OAuth, UI bagus
4. **Logto** — TypeScript, modern UI, free self-hosted
5. **Kanidm** — Rust-based, super lightweight

**Next Decision Point:**
- Option A: Keep Authentik, implement hacky workaround (custom template redirect)
- Option B: Migrate to Zitadel (verify licensing first)
- Option C: Migrate to Casdoor / Logto (need quick eval)
- Option D: Balik Dex + implement separate user management UI (e.g., React dashboard)

**Action:** Tentuin approach sebelum lanjut implementation
